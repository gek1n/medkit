Продуктове бачення

MedKit — мобільний застосунок для управління ліками всієї родини. Користувач за 5 хвилин налаштовує розклад прийомів для себе та рідних, отримує розумні нагадування й контролює дотримання режиму лікування одним поглядом на дашборд.

Ключова цінність

Одна людина (доросла дитина, опікун) налаштовує все за 5 хвилин і витрачає 30 секунд на день, щоб переконатися що мама й тато прийняли таблетки. AI та голосове управління роблять все інше.

⚠️ Головний конкурент — MyTherapy ($14.99/міс без сімейного плану). Наше вікно: сімейний сценарій + залежні профілі без реєстрації + голосове управління + доступна ціна + ринок UA/EU.

Функции

Управління ліками

  • Додавання ліків: назва, форма випуску (таблетки / краплі / сироп / мазь / інгалятор / ін'єкції), дозування
  • Гнучкий розклад прийому: кілька прийомів на день, час відносно їжі (до / після / під час / незалежно)
  • Повтор та інтервал: щодня / через день / певні дні тижня / кожні N днів / циклом (N днів пити — M днів перерва)
  • Тривалість курсу: конкретна кількість днів або постійно
  • Фото упаковки та таблетки — щоб не переплутати, відображається в сповіщеннях
  • AI-скан рецепта — один або кілька препаратів одночасно, batch-wizard для налаштування часу
  • Моніторинг побічних ефектів: AI читає інструкцію і пропонує чіпи симптомів для відстеження

Залишок ліків

  • Таблетки/капсули: точний лічильник, прогрес-бар, прогноз «вистачить на N днів»
  • Краплі/сироп/суспензія: візуальний флакон, оцінка залишку на погляд (75% / 40% / 15%), дата відкриття
  • Мазь/крем/інгалятор: приблизний відсоток використання
  • Кнопки «Поповнити упаковку» / «Відкрив новий флакон» / «🛒 Купити» (партнерське посилання на аптеку)
  • AI-сповіщення за 3–5 днів до закінчення

Активність та фізнавантаження

  • Типи: прогулянка, зарядка, ЛФК, тренування, йога, велосипед та інші
  • Кілька занять на день з окремим розкладом (Заняття 1 / Заняття 2 / ...)
  • Дні тижня, тривалість, нагадування
  • Голосова команда: «Додай прогулянку щодня о 18:00»

Щоденник самопочуття

  • Зрізи 2–3 рази на день за розкладом
  • Вибір настрою: 5 емодзі
  • Чіпи симптомів — автоматично з побічних ефектів поточних ліків
  • Текстовий або голосовий коментар з транскрипцією (3 стани: запис → транскрипція → підтвердження)
  • Для залежних профілів: блок самопочуття прямо на головному екрані + кнопка «Надіслати опікуну»
  • Історія: графік настрою, хронологічний список зрізів
  • AI-кореляції: зв'язок між пропусками таблеток і симптомами
  • Експорт даних для лікаря (PDF)

Сімейне управління

  • До 10 профілів на сімейному тарифі
  • Два типи членів сім'ї: повноцінний акаунт (запрошення) або залежний профіль (без реєстрації)
  • Канали сповіщень для кожного: Push / Telegram-бот / SMS
  • Фото ліків відображається в сповіщенні — літній родич не переплутає
  • Єдиний дашборд сім'ї зі статусами: ✓ прийняв / ⚠ пропустив / ⏳ ще не час
  • Кнопка «Нагадати» — ручне повторне сповіщення одним тапом
  • Самопочуття залежного профілю надходить опікуну як сповіщення

Скан рецепта — кілька препаратів

  • OCR + GPT-4o Vision розпізнає всі препарати з рецепта одночасно
  • Список з чекбоксами — користувач вибирає що додавати
  • Сумнівні позиції (нерозбірливий почерк) позначені ⚡ з варіантами для уточнення
  • Batch-wizard: налаштування часу для кожного препарату по черзі
  • Кнопка «Застосувати ці самі часи до всіх» — для типових схем прийому
  • Оригінал рецепта зберігається у профілі

Запис до лікаря

  • На першому етапі — просте нагадування без інтеграцій з клініками
  • Форма: для кого, який лікар/процедура, де, дата і час, нагадати заздалегідь, нотатка
  • Опціонально: прикріпити PDF-виписку з ліками та симптомами за 30 днів
  • AI нагадує коли курс закінчується і пропонує записатися — без медичних порад

Залежні профілі (без реєстрації)

Залежний профіль — це профіль людини (батьки, дитина) всередині акаунту основного користувача. Людина не реєструється і не має власного акаунту. Усіма ліками та розкладом керує опікун.

✅ Рішення прийнято: залежні профілі — ключова диференціація від конкурентів. MyTherapy, Medisafe, Round Health не мають цієї функції.

  • Прогрес-бар «прийнято X з Y ліків»
  • Блок самопочуття: 5 емодзі + чіпи симптомів + голосова нотатка
  • Кнопка «Надіслати самопочуття» — опікун отримує сповіщення
  • Контакт опікуна з кнопкою «Зателефонувати»
  • Немає: нижньої навігації, доступу до аналітики, можливості змінювати розклад

Голосове управління

Голосове управління — окремий екран і FAB-кнопка на головному екрані. Реалізація: нативний Speech API (безкоштовно) → текст → парсинг інтентів → підтвердження перед дією. Жодна дія не виконується без підтвердження користувача.

✅ Рішення прийнято: особливо цінно для літніх користувачів залежних профілів — замість введення тексту просто говорять про симптоми.

AI-функції

⚠️ Важливий принцип: застосунок не дає медичних порад і не ставить діагнозів. Усі AI-функції допомагають фіксувати та структурувати інформацію для передачі лікарю. У кожному AI-блоці явно зазначено: «Уточніть у свого лікаря» / «Не є медичною рекомендацією». Це і юридична вимога, і UX-рішення: користувач довіряє застосунку саме тому, що він не намагається замінити лікаря.

1. Скан рецепта — один або кілька препаратів

Фото рецепта → GPT-4o Vision розпізнає всі препарати, дозування, частоту, тривалість. Якщо в рецепті кілька ліків — показує список з чекбоксами. Сумнівні позиції позначаються ⚡ з варіантами для уточнення. Після вибору — batch-wizard для налаштування часу. Дані беруться з рецепта — не є рекомендацією застосунку.

2. Підказки щодо розкладу

На основі інструкції до препарату застосунок пропонує час прийому (після їжі, інтервал між дозами). Формулювання в UI: «З інструкції: ... Уточніть у свого лікаря». Прямих порад щодо зміни дозування немає.

3. Чіпи симптомів з інструкції

Застосунок читає список побічних ефектів з інструкції і пропонує їх як швидкі чіпи у формі самопочуття. Користувач сам відмічає що відчуває — дані накопичуються для лікаря. Застосунок не інтерпретує симптоми.

4. Фіксація зв'язку пропусків і симптомів

Застосунок відображає хронологію: пропуск таблетки + симптоми того самого дня. Не робить висновків — тільки показує дані у зручному вигляді для обговорення з лікарем. Формулювання: «Ці дані допоможуть лікарю на наступному візиті».

5. Парсинг голосових команд

Голос → текст (нативний Speech API) → розпізнавання інтенту (локальні правила + Claude API для складних фраз). Застосунок завжди показує що зрозумів і чекає підтвердження — жодна дія не виконується автоматично. Особливо цінно для літніх користувачів залежних профілів.

6. Проактивні нагадування (не поради)

Застосунок нагадує про факти: «Ліки закінчуються через 4 дні», «Курс Еналаприлу завершується — можливо, варто уточнити у лікаря продовження». Формулювання завжди м'яке: «можливо», «варто уточнити» — без тверджень.

7. PDF-звіт для лікаря

Структурований вивід фактичних даних: список ліків, дотримання режиму, щоденник симптомів за період. Лікар сам робить висновки — застосунок тільки агрегує дані. Прикріплюється до запису на прийом одним тапом.

Швидкий старт (онбординг)

Майстер з 6 кроків (~5 хвилин). Після завершення користувач одразу бачить налаштований дашборд.

1
Кого налаштовуємо — Я / Мама / Тато / Дитина / Партнер (можна всіх одразу)
2
Сім'я — тип профілю (повний акаунт або залежний), ім'я, аватар, спосіб доступу
3
Ліки — скан рецепта (один або кілька препаратів) або введення вручну, перемикач по профілях
4
AI будує розклад — підтвердити або змістити час
5
Активність + самопочуття — увімкнути одним тоглом
6

Flutter — ключові пакети

  • http — запити до PHP API
  • speech_to_text — голосове управління (нативний Speech API, безкоштовно)
  • camera — скан рецептів
  • local_auth — Face ID / Touch ID для залежних профілів
  • flutter_local_notifications — локальні нагадування про ліки (без сервера)
  • firebase_messaging — FCM push для сімейних сповіщень
  • isar — локальна БД для офлайн-режиму
  • riverpod — стейт-менеджмент
  • go_router — навігація

Структура PHP API

  • api/auth.php — реєстрація, JWT, залежні профілі, одноразовий код входу
  • api/meds.php — ліки, розклад, прийоми
  • api/family.php — сім'я, права доступу, налаштування членів
  • api/wellbeing.php — самопочуття, симптоми
  • api/appointments.php — записи до лікаря
  • api/scan.php — проксі до GPT-4o Vision, перевірка ліміту сканів
  • api/voice.php — проксі до Claude API для складних команд
  • api/push.php — відправка FCM push для сімейних нагадувань
  • cron/reminders.php — кожні 5 хв: шукає прострочені прийоми і надсилає push опікуну

Аутентифікація та безпека

  • JWT токени: access token 15 хв + refresh token 30 днів (httpOnly cookie). Stateless — сервер не зберігає сесії
  • Паролі: bcrypt, ніколи не зберігаються у відкритому вигляді
  • Код входу залежного профілю: bcrypt-хеш у БД, plain текст тільки при генерації
  • SQL: виключно prepared statements через PDO — захист від SQL injection
  • IDOR: перед кожною операцією перевірка що ресурс належить поточному користувачу
  • Rate limiting: 5 спроб авторизації / хв на IP; 10 сканів рецептів / день на акаунт
  • Input validation: whitelist дозволених полів при кожному запиті, ніяких масових присвоювань

API ключі (GPT-4o, Claude)

Ключі — тільки на сервері, Flutter їх ніколи не бачить

Flutter надсилає запит на свій PHP сервер → сервер звертається до OpenAI / Claude від свого імені. Ключі зберігаються у .env файлі поза webroot і закриті через .htaccess. При компрометації — міняється тільки .env, застосунок не перевидається.
